Book JSON Import — Yaklaşım Seçenekleri

Admin panele JSON kitap import özelliği için 3 farklı mimari yaklaşım

A

Tek Ekran — All-in-One

Tek bir BookJsonImportScreen içinde her şey: dosya yükleme/yapıştırma, validation, preview, import.

Pros

  • Tek dosya, basit navigasyon
  • Mevcut import pattern'lerine en yakın (user_import, vocab_import)
  • Hızlı geliştirme

Cons

  • Ekran karmaşıklaşabilir (validation + preview + import)
  • State yönetimi tek widget'ta yoğunlaşır
B

Stepper / Wizard

3 adımlı wizard: 1) JSON gir (dosya/yapıştır) → 2) Validation + Preview → 3) Import sonucu. Flutter Stepper widget kullanır.

Pros

  • Her adım ayrı ve net
  • Validate-first akışına doğal uyum
  • Kullanıcı nerede olduğunu bilir

Cons

  • Daha fazla kod
  • Flutter Stepper widget bazen kısıtlayıcı
C

Tek Ekran + Dialog

Ana ekranda JSON giriş + validation. Import'u mevcut CsvImportDialog pattern'ine benzer bir dialog'da çalıştır (progress bar + sonuç).

Pros

  • Mevcut admin panel pattern'iyle tutarlı
  • Dialog import sırasında sayfadan çıkmayı engeller
  • Preview ana ekranda, import dialog'da — sorumluluk ayrımı iyi

Cons

  • Dialog içi progress için ayrı state gerekir

Önerim: B (Stepper) — Validate-first kararıyla en doğal eşleşen yaklaşım. Her adım izole ve test edilebilir.
